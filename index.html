Overview

This is CRESS-GLUE, a GLUE project designed to support comparative genomic and evolutionary analysis of circular Rep-encoding single-stranded DNA (CRESS DNA) viruses.

GLUE is a flexible software environment for working with molecular sequence data derived from viruses. GLUE projects are effectively mini-databases that capture information about specific viruses or virus groups.

CRESS-GLUE contains aligned, annotated genome sequence data for CRESS DNA viruses and endogenous viral elements (EVEs) derived from these viruses.

GLUE project

On computers with GLUE installed, the CRESS-GLUE project can be instantiated by navigating to the project folder, initiating GLUE, and issuing the following command in the GLUE shell: 

  Mode path: /
  GLUE> run file cressProject.glue

What is a GLUE project?

GLUE is a flexible bioinformatics environment that enables the rapid development of diverse sequence data resources for different viruses. The GLUE “engine” is an open, integrated software toolkit that provides functionality for storage and interpretation of sequence data. GLUE supports the development of “projects” comprised of data (nucleic acid sequences plus any other kind of associated data) and program logic. Projects are loaded into the GLUE engine so that the complex relationships between project data items are captured in a relational database.

The GLUE software “engine” provides a set of tools for manipulating and interrogating project data. The core database schema can be extended to accommodate the idiosyncrasies of different projects. GLUE also provides a scripting layer for developing custom analysis tools.

Hosting of GLUE projects in an online version control system (e.g. GitHub) provides a mechanism for their stable, collaborative development.

What does building the CRESS-GLUE project offer?

The CRESS-GLUE project not only contains data items required for comparative genomic analysis of these viruses (i.e. virus genome sequences, multiple sequence alignments, genome feature annotations, and other sequence-associated data), it also uses a relational database to represent the semantic relationships between these data items.

Furthermore, via the GLUE engine (a platform-independent software environment) provides the means to recapitulate all of the analyses performed in our study.
